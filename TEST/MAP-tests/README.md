# Map File Verification Tests

## Purpose

Verify that `.map` files generated by the `-m` flag accurately reflect compiled object structure by cross-referencing against:
1. **Listing files (`.lst`)** - Generated by `-l` flag, contains symbol table and hex dump
2. **Expected definitions (`expected.json`)** - Ground truth derived from source file analysis

## Methodology

1. Define test objects with known, predictable structures
2. Capture expectations in `expected.json` for each test
3. Compile with both `-l -m` flags
4. Run `verify-map.ts` to cross-check all three sources (source expectations, listing, map)

## Test Cases

### test1-simple (Basic Parent/Child)
- **Structure**: 2 objects, depth 1
- **Focus**: Basic parent/child relationship, simple method signatures
- **Methods**: `PUB go()`, `PUB add(a, b) : sum`
- **VAR**: LONG and BYTE variables
- **DAT**: Single LONG in child

### test2-deep (Deep Nesting)
- **Structure**: 3 objects, depth 2 (top -> mid -> leaf)
- **Focus**: Deep object hierarchy, multiple return values, local variables
- **Methods**: `PUB run() : x, y`, `PRI helper(p1, p2, p3)`, methods with locals
- **VAR**: Mixed types at each level
- **DAT**: Variables at leaf level

### test3-wide (Breadth with PASM)
- **Structure**: 4 objects, breadth 3 (top with 3 children)
- **Focus**: Sibling objects, PASM labels, DAT varieties
- **Methods**: Mix of all signature forms
- **VAR**: Different types per child
- **DAT**: PASM code with labels in one child

## What We Verify

### Symbol Counts
- Number of DAT symbols in map matches listing
- Number of VAR symbols in map matches listing
- Number of methods in map matches listing

### Address/Offset Consistency
- DAT symbol offsets in map match VALUE field in listing
- VAR symbol offsets in map match VALUE field in listing
- Method entry points in map match VALUE field in listing

### Size Totals
- `OBJ bytes` in listing matches executable size in map
- `VAR bytes` in listing matches variable size in map

### Object Hierarchy
- Object count in map matches expected
- Object names correlate correctly with file IDs

### PASM Labels
- PASM labels appear in both DAT Sections and PASM Labels sections
- COG addresses calculated correctly (DAT offset / 4)

### Method Signatures (Future Enhancement)
- Parameter counts match source
- Return value counts match source
- Local variable counts match source
